DUNGEON & FLOOR SYSTEM — Reference
====================================

GLOBAL VARIABLES
----------------

global.dungeonlist[]
    Array of dungeon structs loaded from DungeonImport.csv by InitDungeons().
    Each entry defines a full chapter with its own enemy pool, boss, rewards, and overrides.

global.dungeon
    Integer index into global.dungeonlist[]. The current dungeon/chapter the player is in.

global.dungeonFloor
    1-based integer. The player's current floor within the current dungeon.
    Resets to 1 at the start of each dungeon.

global.dungeonTroops
    Array of troop arrays loaded from the current dungeon's troop CSV.
    Same format as the old global.troops — each entry is an array of monster name strings.
    Reloaded by StartDungeon() and LoadGame().

global.floorChallenges[]
    Array of challenge structs for the current floor. Generated by GenerateFloor().
    Each struct:
        type           "combat", "boss", or "puzzle" (puzzle is a future stub)
        troop          Array of monster name strings for this encounter
        completed      Bool — set to true by ProcessPostBattleQueue when defeated
        unique         Bool — if true, tracked in defeatedMiniBosses when completed
        override_name  String identifier for unique tracking (semicolon-joined troop names)

global.onFloor
    Bool. Set to true once any challenge on the current floor has been started.
    Reset to false by GenerateFloor() when a new floor is entered.

global.defeatedMiniBosses[]
    Array of override_name strings. Tracks which unique encounters have been
    defeated during the current dungeon run. Cleared by StartDungeon().
    Prevents the same unique override from appearing again.

global.floor
    Integer. Total floors cleared across all dungeons (kept for save/stats).
    Still incremented by NextFloor() alongside dungeonFloor.


DUNGEON STRUCT (global.dungeonlist[] entries)
---------------------------------------------

{
    name             String — display name (e.g. "Mercury Lighthouse")
    floors           Number — how many floors before the boss floor
    background       Sprite asset index — combat background for this dungeon
    troop_csv        String — filename of the troop CSV in datafiles/
    boss             String — monster name for the final boss encounter
    curse_chance     Number 0-100 — percent chance of DifficultyUp() per floor
    rewards          Struct — keys are floor number strings, values are reward type strings
                     e.g. { "3": "levelup", "5": "djinn", "7": "levelup" }
    towns            Array of strings — town names accessible from this dungeon (future)
    min_challenges   Number — minimum challenges generated per floor
    max_challenges   Number — maximum challenges generated per floor
    overrides        Array of override structs (populated from DungeonOverrides.csv)
}


OVERRIDE STRUCT (inside dungeon.overrides[])
---------------------------------------------

{
    dungeon          String — dungeon name this override belongs to
    floor_num        Number — which floor this override triggers on
    type             String — "combat", "boss", or "puzzle"
    troops_a         Array of monster name strings — primary troop option
    troops_b         Array of monster name strings — alternative (picked 50/50 if non-empty)
    solo             Bool — if true, this is the ONLY challenge on this floor
    unique           Bool — if true, only appears once per dungeon run
}


CSV FORMAT: DungeonImport.csv
------------------------------

Header row:
    name,floors,background,troop_csv,boss,curse_chance,reward_schedule,towns,min_challenges,max_challenges

Column details:
    name             Display name of the dungeon
    floors           Number of floors before the boss (boss is floor N+1)
    background       Sprite asset name (must exist in GMS2 project, e.g. Mercury_Lighthouse)
    troop_csv        Filename of the enemy pool CSV in datafiles/ (e.g. TroopImport.csv)
    boss             Monster name string matching an entry in global.monsterlist
    curse_chance     0-100, percentage chance per floor to call DifficultyUp()
    reward_schedule  Semicolon-separated floor:reward pairs
                     Valid rewards: "levelup", "djinn"
                     Example: 3:levelup;5:djinn;7:levelup
                     Means floor 3 gives level up, floor 5 gives djinn draft, floor 7 gives level up
    towns            Semicolon-separated town names (future feature, can leave empty)
    min_challenges   Minimum number of challenges per non-override floor (1-3 typical)
    max_challenges   Maximum number of challenges per non-override floor

Example row:
    Mercury Lighthouse,8,Mercury_Lighthouse,TroopImport.csv,Saturos,40,3:levelup;5:djinn;7:levelup,Vault;Imil,1,3


CSV FORMAT: DungeonOverrides.csv
---------------------------------

Header row:
    dungeon,floor,type,troops_a,troops_b,solo,unique

Column details:
    dungeon          Dungeon name — must match a name in DungeonImport.csv exactly
    floor            Floor number within that dungeon (1-based)
    type             "combat", "boss", or "puzzle"
    troops_a         Semicolon-separated monster names for the encounter
                     Example: Star Magician;Anger Ball;Refresh Ball
    troops_b         Alternative troop list (50/50 chance vs troops_a). Leave empty for no alt.
    solo             TRUE or FALSE — if TRUE, no other challenges spawn on this floor
    unique           TRUE or FALSE — if TRUE, only appears once per dungeon run

Example row:
    Mercury Lighthouse,4,combat,Mimic,,FALSE,TRUE


TROOP CSV FORMAT (e.g. TroopImport.csv, TroopMercury.csv)
-----------------------------------------------------------

Header row:
    number,Entries,,,,,,,

Each subsequent row:
    Column 0: number of monsters in this troop
    Columns 1-N: monster name strings (must match entries in global.monsterlist)

Pad unused columns with empty values. The number of header commas should accommodate
the largest troop size.

Example:
    number,Entries,,,,,,,
    5,Ooze,Gnome,Siren,Ooze,Ooze,,,
    4,Gnome,Lizard Man,Mauler,Gnome,,,,
    1,Mimic,,,,,,,


FLOW OVERVIEW
--------------

1. Game start: InitTroops() + InitDungeons() load all data
2. objBegin click: StartDungeon(0) sets up dungeon 1, calls GenerateFloor()
3. GenerateFloor() populates global.floorChallenges with combat encounters
4. Player picks a challenge → StartCombat(challenge.troop) begins combat
5. Victory → ProcessPostBattleQueue marks challenge complete
6. All challenges done → NextFloor() checks rewards/curses, advances dungeonFloor
7. If dungeonFloor > dungeon.floors → boss floor (single boss challenge)
8. After boss defeated → CompleteDungeon() advances to next dungeon
